How to get parentage files for files from common samples and skims

  • Get your parentage for your common sample read this D0News  by Marco Verzocchi (which replaces the older D0News)

    /home/mverzocc/scripts/genLBNtables -skim <skimname> -pass <passname> -filelist <listfiles>    [-format TMB -type LM ]

  • This will create a subdirectory ./<skimname> which will contain the parentage path (1 file per each input file).

  • For the -pass option use p17pass1 for unfixed P17 data (p17.03.03 and p17.05.01), p14pass2 for the pass2 fixed P14 data (p14.fixtmb2.02) and p14pass1 for pass1 P14 data (a mixture of p14.06.00, p14.05.02 unfixed data and data fixed with p14.fixtmb.01, p14.fixtmb.02 and p14.06.00)
  • You need to provide a list of files through the -filelist option, these are the files in the common sample group skim which you have used in your analysis (one file per line). Alternatively you can use a SAM dataset definition with the -defname option. If you don't provide neither the -filelist nor the -defname options the script will assume that you have processed all the data in the common sample group skims.
  • In the future the genLBNtables script will also support a list of CAFtrees (or a corresponding SAM dataset definition). If you are providing a list of CAF trees use the -format CAF option (Option not supported as of 8/22/2005, because the CAF trees are not stored in SAM. If you want to calculate the luminosity corresponding to the CAF trees you are including in the analysis use the -format TMB option and pass the list of files used to generate the CAF trees with the -filelist ....... option).
