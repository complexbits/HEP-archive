dumpLBN utility

You can dump some of the contents of an lbn, normally enough to see if it is ok, by invoking the binary dumpLBN.

 

The binary dumpLBN is created when lm_access libs are created. lm_access has to be in your path for this to work - you will need to build it yourself until it gets into the release.

syntax is

dumpLBN <lbn number > [<trigger>]

the default trigger is EM_HI_SH

Example:

prompt>dumpLBN 1712341 zero_bias

dump of lbn 1712341 for trigger zero_bias
Delivered Trigger Status:0 Delivered Luminosity:1058.24
Triggered Trigger Status:1 Triggered Luminosity:0.0003037
Recorded Trigger Status:1 Recorded Luminosity:0.0003037
Trigger has L2?: 0
L1 Accepts: 31 L2 Accepts: 31 L3 Accepts: 31
Reconstructed information
tmb p13.02.00 p13.02.00, keepL2 = 0, status = 1
tmb p13.05.00 p13.05.00, keepL2 = 1, status = 1


This shows that this lbn was recorded properly


prompt> dumpLBN 2880120 mu1ptxatxx_ncu

dump of lbn 2880120 for trigger mu1ptxatxx_ncu

Delivered Trigger Status:0 Delivered Luminosity:425.905

Triggered Trigger Status:1 Triggered Luminosity:0.1249
Recorded Trigger Status:1 Recorded Luminosity:0.1249
Trigger has L2?: 0
L1 Accepts: 5 L2 Accepts: 5 L3 Accepts: 5

If the trigger is not valid for that LBN you don't see any information

prompt> dumpLBN 2880120 EM_HI_SH

dump of lbn 2880120 for trigger EM_HI_SH
Reconstructed information
